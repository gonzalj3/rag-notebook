Yes — your handwritten note from April 1st directly connects. You wrote that the eval pipeline should be built before the generation layer, which aligns with how you should approach hybrid retrieval too. Before optimizing your LLM prompts or generation quality, you should first measure whether your retrieval is finding the right chunks.

Hamel Husain's article reinforces this: he separates RAG evaluation into retrieval metrics (Recall@k, Precision@k, MRR) and generation metrics. The retrieval side — which is where hybrid search lives — should be measured first because if you retrieve wrong chunks, no generation model can produce correct answers.

In practice for your notebook: get hybrid retrieval working, build a small eval set to measure retrieval quality, and only then focus on how well the on-device LLM generates answers from those chunks.
